# 回合制策略H5网页网游 - 整体架构设计

## 一、技术选型

### 1.1 前端技术栈
- **框架**: React 18 + TypeScript
- **构建工具**: Vite 5
- **状态管理**: Zustand (轻量级，适合游戏状态)
- **UI组件库**: Ant Design Mobile (H5适配)
- **网络请求**: Axios + WebSocket (战斗实时推送)
- **游戏渲染**: Canvas 2D (轻量级战斗场景)
- **样式方案**: TailwindCSS + CSS Modules
- **路由**: React Router v6

### 1.2 后端技术栈
- **运行时**: Node.js 20 + TypeScript
- **Web框架**: NestJS (企业级，模块化架构)
- **数据库**: 
  - MySQL 8.0 (主数据存储)
  - Redis 7 (缓存、会话、排行榜)
- **ORM**: Prisma
- **消息队列**: BullMQ (基于Redis，异步任务、跨服通信)
- **实时通信**: Socket.io
- **配置中心**: 本地JSON + 版本管理 (Demo阶段)

### 1.3 部署与运维
- **容器化**: Docker + Docker Compose
- **进程管理**: PM2
- **日志**: Winston + ELK Stack (Demo阶段简化为文件日志)

## 四、核心模块设计

### 4.1 数值养成体系

#### 属性计算模型
- 采用**加法叠加 + 百分比相乘**的混合模型
- 基础属性 → 固定加成 → 百分比加成 → 最终数值
- 所有属性计算统一服务端完成

```typescript
// 属性计算顺序
1. 基础属性 (武将初始)
2. 等级加成 (固定值)
3. 进阶加成 (固定值 + 百分比)
4. 天赋加成 (百分比)
5. 羁绊加成 (百分比)
6. 装备加成 (固定值 + 百分比)
7. 符文加成 (百分比 + 特殊效果)
8. 城池科技加成 (全局百分比)
```

#### 战力计算公式
```
战力 = Σ(攻击属性 * 攻击权重) + Σ(防御属性 * 防御权重) + Σ(生命属性 * 生命权重)
     + 技能战力加成 + 羁绊战力加成
```

### 4.2 战斗系统

#### 战斗流程
1. 战前准备：阵容校验、属性计算、技能预加载
2. 回合制战斗：按速度排序行动，自动释放技能
3. 伤害计算：攻击方属性 vs 防御方属性 → 最终伤害
4. 战斗结算：胜负判定、奖励发放、战报生成

#### 战报系统
- 采用回合制快照式存储
- 每回合记录：行动者、目标、技能、伤害、状态变化
- 战报压缩：差异存储 + Gzip压缩
- 支持回放：按回合逐步播放

### 4.3 赛季与排位体系

#### 赛季类型
- **常规赛季**: 28天一个周期，日常排位赛
- **季后赛**: 赛季末前64名晋级，单败淘汰制
- **跨服联赛**: 每月一次，多服混战

#### 段位系统
```
青铜 → 白银 → 黄金 → 铂金 → 钻石 → 大师 → 王者
   5段    5段     5段     5段     5段    3段    无段
```

#### 赛季结算
1. 赛季结束时间到，冻结排位数据
2. 计算最终排名和段位
3. 发放赛季奖励 (邮件)
4. 段位重置 (基于当前段位软重置)
5. 历史赛季数据归档

### 4.4 城池经营系统

#### 建筑类型
- **主殿**: 决定其他建筑等级上限
- **农田**: 产出粮食
- **银库**: 产出银币
- **兵营**: 提升守军数量
- **校场**: 提升武将经验获取
- **铁匠铺**: 装备强化相关
- **仓库**: 资源存储上限

#### 资源产出机制
- 采用**时间累积**模式
- 服务端定时结算 (每分钟)
- 上线时一次性结算离线收益
- 防刷机制：时间校验、产出上限、异常检测

### 4.5 配置中心

#### 配置类型
- 武将配置 (基础属性、技能、羁绊)
- 装备配置 (属性、套装效果)
- 活动配置 (时间、奖励、规则)
- 赛季配置 (时长、奖励、段位表)
- 数值配置 (战斗公式参数)

#### 版本管理
- 每个配置文件带版本号
- 修改自动备份上一版本
- 支持一键回滚到指定版本
- 灰度发布：按玩家比例/分组逐步生效

## 五、数据库设计概览

### 核心数据表
1. `users` - 用户表
2. `user_heroes` - 用户武将表
3. `user_equipment` - 用户装备表
5. `user_resources` - 用户资源表
6. `user_city` - 用户城池表
7. `arena_ranks` - 排位赛排名表
8. `battle_reports` - 战报表
9. `mails` - 邮件表
10. `config_versions` - 配置版本表

### Redis数据结构
- 用户会话: `session:{userId}` → Hash
- 排行榜: `arena:rank:{season}` → Sorted Set
- 在线玩家: `online:users` → Set
- 资源产出队列: `resource:queue` → List
